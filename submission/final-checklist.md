# Final Checklist (Hackathon Submission)

## Repo readiness
- [ ] `README.md` explains what it is, why it matters, demo flow, how to run, and tech stack
- [ ] Specs exist and match the demo scope (`specs/`)
- [ ] Demo flow and judge script are rehearsed (`product/`)
- [ ] Submission docs are present (`submission/`)

## Demo must-pass
- [ ] Daily Intent can be set/edited quickly
- [ ] Agent Suggest shows a recommendation + 1-line reason
- [ ] Timer start/pause/resume works reliably
- [ ] Ring progress UI updates and is legible
- [ ] Reflection can be submitted; score displayed
- [ ] Telemetry log exists and contains the full demo event sequence

## Tech polish (high leverage)
- [ ] Add a “Reset Demo” button or predictable seed data
- [ ] Ensure empty states are friendly (no crashes if no tasks/intents)
- [ ] Keep font sizes readable on iPhone simulator

## Packaging
- [ ] Record a short backup screen capture (in case of demo hiccup)
- [ ] Add 1 screenshot/GIF to README (optional)
- [ ] Confirm build runs on a clean machine (or at least a clean simulator)
